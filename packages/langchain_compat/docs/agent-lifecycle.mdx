# ChatAgent Lifecycle

Understand how chat agents are created, configured, and managed.

## Creation Patterns

```dart
// Basic
final agent = ChatAgent('openai');

// With model
final agent = ChatAgent('openai:gpt-4o');

// With configuration
final agent = ChatAgent(
  'anthropic:claude-3-5-haiku',
  temperature: 0.7,
  systemPrompt: 'Be concise.',
  tools: [weatherTool],
);

// From provider
final provider = ChatProvider.forName('google');
final agent = ChatAgent.forProvider(provider);
```

## Lazy Initialization

Models are created on first use:

```dart
final agent = ChatAgent('openai');  // Lightweight creation
final result = await agent.run('Hello'); // Model created here
```

## Stateless Design

ChatAgents are stateless - conversation state lives in history:

```dart
final agent = ChatAgent('anthropic');

// Multiple independent conversations
final customerHistory = <ChatMessage>[];
final supportHistory = <ChatMessage>[];

// Same chat agent, different contexts
await agent.run('Customer query', history: customerHistory);
await agent.run('Support ticket', history: supportHistory);
```

## Resource Management

ChatAgents are lightweight and don't need explicit cleanup:

```dart
Future<void> processRequest() async {
  final agent = ChatAgent('google'); // Created
  await agent.run('Process this');
  // Automatically cleaned up when out of scope
}
```

## Testing

Mock chat agents for testing:

```dart
class MockChatAgent implements ChatAgent {
  final List<String> responses;
  int callCount = 0;
  
  @override
  Future<ChatResult<String>> run(prompt, {history}) async {
    return ChatResult(
      output: responses[callCount++],
      messages: [
        ChatMessage.user(prompt),
        ChatMessage.model(responses[callCount-1]),
      ],
    );
  }
}

// In tests
final mockAgent = MockChatAgent(['Response 1', 'Response 2']);
await myService.process(mockAgent);
```

## Performance Tips

1. **Reuse chat agents** for the same configuration
2. **Share history** between related calls
3. **Use appropriate models** for each task
4. **Stream for long responses**
