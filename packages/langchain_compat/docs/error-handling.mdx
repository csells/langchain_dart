# Error Handling

Build robust applications by handling errors gracefully.

## Common Errors

### API Errors
```dart
try {
  await agent.run('Hello');
} on OpenAIClientException catch (e) {
  if (e.statusCode == 429) {
    print('Rate limited');
  } else if (e.statusCode == 401) {
    print('Invalid API key');
  }
}
```

### Network Errors
```dart
try {
  await agent.run('Hello');
} on SocketException {
  print('Network connection failed');
} on TimeoutException {
  print('Request timed out');
}
```

### Tool Errors
```dart
final tool = Tool(
  name: 'api_call',
  description: 'Call external API',
  onCall: (args) async {
    throw ToolException('API unavailable');
  },
);

// Tool errors are passed to the model
final agent = ChatAgent('openai', tools: [tool]);
await agent.run('Use the API'); // Model sees error
```

## Retry Strategy

```dart
Future<ChatResult> runWithRetry(
  ChatAgent agent,
  String prompt, {
  int maxRetries = 3,
}) async {
  for (var i = 0; i < maxRetries; i++) {
    try {
      return await agent.run(prompt);
    } catch (e) {
      if (i == maxRetries - 1) rethrow;
      await Future.delayed(Duration(seconds: 1 << i));
    }
  }
  throw Exception('Max retries exceeded');
}
```

## Provider Fallback

```dart
Future<ChatResult> runWithFallback(String prompt) async {
  final providers = [
    'openai',
    'anthropic',
    'google',
  ];
  
  for (final provider in providers) {
    try {
      final agent = ChatAgent(provider);
      return await agent.run(prompt);
    } catch (e) {
      print('$provider failed: $e');
      if (provider == providers.last) rethrow;
    }
  }
  
  throw Exception('All providers failed');
}
```

## Rate Limiting

Dartantic includes automatic retry for rate limits:

```dart
// Automatic retry with exponential backoff
final agent = ChatAgent('openai');
await agent.run('Hello'); // Retries on 429
```

See [RetryHttpClient](https://github.com/davidmigloz/langchain_dart/blob/main/packages/langchain_compat/lib/src/http/retry_http_client.dart) for details.

## Best Practices

1. **Log errors** for debugging
2. **Provide user-friendly messages**
3. **Implement fallbacks** for critical paths
4. **Respect rate limits**
5. **Handle tool errors gracefully**
