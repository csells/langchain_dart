# Message API

Dartantic provides two complementary ways to work with conversations: the high-level Agent API for most use cases, and the lower-level Message API for advanced scenarios requiring fine-grained control.

## Agent API (Recommended)

For most use cases, use the Agent API which separates conversation history from new user input:

```dart
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  final agent = ChatAgent('openai:gpt-4o-mini');
  
  // Simple usage - Agent handles message construction
  final result = await agent.run('Hello!');
  print('Agent: ${result.output}');
  
  // With conversation history
  final history = [ChatMessage.system('You are helpful')];
  final result2 = await agent.run('Hello!', history: history);
  print('Agent: ${result2.output}');
}
```

## Direct Message Construction

For advanced use cases, you can work directly with the Message/Part model:

### Basic Message Creation

```dart
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  // Creating messages manually
  final systemMsg = ChatMessage.system('You are a helpful assistant');
  final userMsg = ChatMessage.user('Hello!');
  final modelMsg = ChatMessage.model('Hi! How can I help you today?');
  
  print('System: ${systemMsg.text}');
  print('User: ${userMsg.text}');
  print('Assistant: ${modelMsg.text}');
}
```

### Multi-Part Messages

Messages can contain multiple parts including text, images, files, and tool interactions:

```dart
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  // Messages with multiple parts
  final multipartMsg = ChatMessage(
    role: MessageRole.user,
    parts: [
      TextPart('What is in this image?'),
      DataPart(bytes: imageBytes, mimeType: 'image/png'),
    ],
  );
  
  // Process the message
  final agent = ChatAgent('openai:gpt-4o');
  final response = await agent.run('', history: [multipartMsg]);
  print('Response: ${response.output}');
}
```

### File and Media Handling

Attach various types of media to messages:

```dart
import 'dart:io';
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  // Image from file
  final imageFile = File('path/to/image.jpg');
  final imageBytes = await imageFile.readAsBytes();
  
  final imageMessage = ChatMessage(
    role: MessageRole.user,
    parts: [
      TextPart('Analyze this image'),
      DataPart(bytes: imageBytes, mimeType: 'image/jpeg'),
    ],
  );
  
  // PDF document
  final pdfFile = File('path/to/document.pdf');
  final pdfBytes = await pdfFile.readAsBytes();
  
  final documentMessage = ChatMessage(
    role: MessageRole.user,
    parts: [
      TextPart('Summarize this document'),
      DataPart(bytes: pdfBytes, mimeType: 'application/pdf'),
    ],
  );
  
  // Process messages
  final agent = ChatAgent('google:gemini-2.0-flash'); // Good multimodal support
  final result = await agent.run('', history: [imageMessage]);
  print(result.output);
}
```

## Tool Interactions

Handle tool calls and results with explicit message construction:

```dart
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  // Tool call message from the model
  final toolCall = ChatMessage(
    role: MessageRole.model,
    parts: [
      TextPart('Let me check the weather for you.'),
      ToolPart.call(
        id: 'call_123',
        name: 'get_weather',
        arguments: {'location': 'Portland'},
      ),
    ],
  );
  
  // Tool result message from the user/system
  final toolResult = ChatMessage(
    role: MessageRole.user,
    parts: [
      ToolPart.result(
        id: 'call_123',
        name: 'get_weather',
        result: '72°F, sunny',
      ),
    ],
  );
  
  // Continue the conversation
  final history = [toolCall, toolResult];
  final agent = ChatAgent('openai:gpt-4o-mini');
  final response = await agent.run('Great! What should I wear?', history: history);
  print(response.output);
}
```

## Message Roles

Dartantic supports four message roles:

```dart
enum MessageRole {
  system,   // System instructions and behavior
  user,     // User input and tool results
  model,    // Model responses and tool calls
}
```

### System Messages

System messages provide instructions and context:

```dart
final systemMessage = ChatMessage.system(
  'You are a helpful assistant that responds concisely. '
  'Always ask follow-up questions to better understand user needs.'
);
```

### User Messages

User messages contain user input and tool results:

```dart
final userMessage = ChatMessage.user('What is the weather like today?');

// User message with tool result
final toolResultMessage = ChatMessage(
  role: MessageRole.user,
  parts: [
    ToolPart.result(
      id: 'weather_call_123',
      name: 'get_weather',
      result: 'Today: 75°F, partly cloudy',
    ),
  ],
);
```

### Model Messages

Model messages contain AI responses and tool calls:

```dart
final modelResponse = ChatMessage.model('The weather today is quite pleasant!');

// Model message with tool call
final modelWithTool = ChatMessage(
  role: MessageRole.model,
  parts: [
    TextPart('I need to check the current weather.'),
    ToolPart.call(
      id: 'weather_123',
      name: 'get_weather',
      arguments: {'location': 'San Francisco'},
    ),
  ],
);
```

## Part Types

Messages are composed of different part types:

### TextPart
```dart
final textPart = TextPart('Hello, world!');
```

### DataPart
```dart
// Binary data with MIME type
final dataPart = DataPart(
  bytes: fileBytes,
  mimeType: 'image/jpeg',
);
```

### ToolPart
```dart
// Tool call
final toolCall = ToolPart.call(
  id: 'unique_id',
  name: 'function_name',
  arguments: {'param': 'value'},
);

// Tool result
final toolResult = ToolPart.result(
  id: 'unique_id',
  name: 'function_name',
  result: 'Function output',
);
```

## Message Utilities

### Accessing Message Content

```dart
final message = ChatMessage.user('Hello with image');
message.parts.add(DataPart(bytes: imageBytes, mimeType: 'image/png'));

// Get text content
final text = message.text; // 'Hello with image'

// Get all tool calls
final toolCalls = message.toolCalls; // List<ToolPart>

// Check message type
if (message.hasText) {
  print('Message has text');
}

if (message.hasMedia) {
  print('Message has media attachments');
}

if (message.hasToolCalls) {
  print('Message has tool calls');
}
```

### Message Filtering

```dart
final conversation = <ChatMessage>[
  ChatMessage.system('You are helpful'),
  ChatMessage.user('Hello'),
  ChatMessage.model('Hi there!'),
  ChatMessage.user('How are you?'),
];

// Filter by role
final userMessages = conversation.where((m) => m.role == MessageRole.user);
final modelMessages = conversation.where((m) => m.role == MessageRole.model);

// Filter by content type
final textMessages = conversation.where((m) => m.hasText);
final mediaMessages = conversation.where((m) => m.hasMedia);
```

## Advanced Message Patterns

### Conversation Templates

```dart
class ConversationTemplate {
  static List<ChatMessage> createSupportConversation() {
    return [
      ChatMessage.system(
        'You are a helpful customer support agent. '
        'Be polite, professional, and solution-oriented.'
      ),
      ChatMessage.user('I need help with my account'),
      ChatMessage.model(
        'I\'d be happy to help you with your account. '
        'Could you please describe the specific issue you\'re experiencing?'
      ),
    ];
  }
}
```

### Message Builders

```dart
class MessageBuilder {
  final List<Part> _parts = [];
  MessageRole _role = MessageRole.user;
  
  MessageBuilder text(String content) {
    _parts.add(TextPart(content));
    return this;
  }
  
  MessageBuilder image(List<int> bytes, String mimeType) {
    _parts.add(DataPart(bytes: bytes, mimeType: mimeType));
    return this;
  }
  
  MessageBuilder role(MessageRole role) {
    _role = role;
    return this;
  }
  
  ChatMessage build() {
    return ChatMessage(role: _role, parts: _parts);
  }
}

// Usage
final message = MessageBuilder()
  .text('What do you see in this image?')
  .image(imageBytes, 'image/jpeg')
  .role(MessageRole.user)
  .build();
```

## Best Practices

### Message Construction
1. **Use Agent API** for most use cases - simpler and less error-prone
2. **Use Message API** when you need fine-grained control over conversation flow
3. **Include MIME types** for all binary data
4. **Keep tool IDs unique** within a conversation

### Performance
1. **Batch media uploads** when possible
2. **Compress images** before sending to reduce latency
3. **Use appropriate models** for multimodal content (Gemini, GPT-4V)

### Error Handling
1. **Validate MIME types** before creating DataParts
2. **Handle tool errors** gracefully in ToolPart results
3. **Check message size limits** for your chosen provider
