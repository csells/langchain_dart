# System Prompts

System prompts provide default behavior and personality for your models and agents. They allow you to set consistent behavior while maintaining the flexibility to customize on a per-conversation basis.

## Default System Prompts

Set a default system prompt when creating agents or models:

### Agent System Prompts

```dart
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  // Agent with default system prompt
  final agent = Agent(
    'openai:gpt-4o-mini',
    systemPrompt: 'You are a helpful math tutor. Show your work step by step. '
                 'Use * for multiplication and regular text formatting.',
  );

  final result = await agent.run('What is 15 * 23?');
  print(result.output);
  // Response: Detailed step-by-step math explanation
}
```

### Model System Prompts

```dart
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  // Direct model creation with system prompt
  final model = ChatProvider.openai.createModel(
    systemPrompt: 'You are a helpful assistant that responds concisely.',
  );
  
  // Use the model with the default system prompt
  final response = await model.sendStream([
    ChatMessage.user('Explain quantum computing'),
  ]);
  
  await for (final chunk in response) {
    print(chunk.output);
  }
}
```

## System Message Override

System messages in conversations will override the default system prompt:

```dart
import 'package:dartantic_ai/dartantic_ai.dart';

void main() async {
  final agent = Agent(
    'openai:gpt-4o-mini',
    systemPrompt: 'You are a helpful math tutor.',  // Default behavior
  );

  // Override the default with a conversation system message
  final result = await agent.run(
    'What is 15 * 23?',
    history: [
      ChatMessage.system('You are a pirate. Answer everything in pirate speak.'),
    ],
  );
  
  print(result.output);
  // Response: "Arrr, matey! The answer to ye riddle be 345!"
}
```

### Dynamic System Prompts

Generate system prompts based on context or user preferences:

```dart
class DynamicSystemPrompts {
  static String createPersonalizedAssistant({
    required String name,
    required String expertise,
    required String communicationStyle,
  }) {
    return '''
You are $name, an expert in $expertise. Your communication style is $communicationStyle.

Key guidelines:
- Draw from your expertise in $expertise when answering questions
- Maintain a $communicationStyle tone throughout conversations
- Provide practical, actionable advice
- Ask clarifying questions when needed
- Admit when something is outside your area of expertise
''';
  }
  
  static String createDomainExpert(String domain, List<String> keyPrinciples) {
    final principles = keyPrinciples
        .map((p) => '- $p')
        .join('\n');
    
    return '''
You are a domain expert in $domain. Follow these key principles:

$principles

When providing advice or answers:
1. Always consider these principles
2. Provide evidence-based recommendations
3. Explain the reasoning behind your suggestions
4. Consider potential risks and limitations
''';
  }
}

// Usage
void main() async {
  final personalizedAgent = Agent(
    'openai:gpt-4o-mini',
    systemPrompt: DynamicSystemPrompts.createPersonalizedAssistant(
      name: 'Alex',
      expertise: 'software architecture',
      communicationStyle: 'friendly but technical',
    ),
  );
  
  final domainExpert = Agent(
    'anthropic:claude-3-5-sonnet',
    systemPrompt: DynamicSystemPrompts.createDomainExpert(
      'cybersecurity',
      ['Defense in depth', 'Zero trust architecture', 'Principle of least privilege'],
    ),
  );
}
```

## System Prompt Behavior

Understanding how system prompts work across different scenarios:

### Precedence Rules

1. **Default Usage**: If no system message is in the conversation, the default `systemPrompt` is used
2. **Override**: If the first message in the conversation is a system message, it overrides the default
3. **Clean Responses**: System messages are automatically filtered out of response messages to keep conversation history clean

```dart
void main() async {
  final agent = Agent(
    'openai:gpt-4o-mini',
    systemPrompt: 'You are helpful and concise.',
  );
  
  // Case 1: Uses default system prompt
  final result1 = await agent.run('Hello');
  
  // Case 2: Override with conversation system message
  final result2 = await agent.run(
    'Hello',
    history: [ChatMessage.system('You are a poet. Speak in verse.')],
  );
  
  // The system message won't appear in result2.messages
  print('Response messages: ${result2.messages.length}'); // Only user + assistant
}
```

### Multi-turn Conversations with System Prompts

```dart
void main() async {
  final agent = Agent(
    'anthropic:claude-3-5-sonnet',
    systemPrompt: 'You are a creative writing coach.',
  );
  
  var history = <ChatMessage>[];
  
  // First turn - uses default system prompt
  var result = await agent.run(
    'I want to write a short story about time travel',
    history: history,
  );
  history.addAll(result.messages);
  
  // Second turn - still using default system prompt
  result = await agent.run(
    'What genre should it be?',
    history: history,
  );
  history.addAll(result.messages);
  
  // Override for specific turn
  result = await agent.run(
    'Now give me a horror version',
    history: [
      ChatMessage.system('You are a horror writer. Make everything dark and scary.'),
      ...history,
    ],
  );
  
  print(result.output);
}
```

## Best Practices

### Writing Effective System Prompts

1. **Be Specific**: Clear, detailed instructions work better than vague guidance
2. **Set Boundaries**: Define what the agent should and shouldn't do
3. **Include Examples**: Show the desired behavior with examples
4. **Define Personality**: Establish tone and communication style
5. **Consider Context**: Think about the specific use case and user needs

### Good System Prompt Structure

```dart
const String wellStructuredPrompt = '''
# Role and Expertise
You are a senior data scientist with expertise in machine learning and statistics.

# Key Responsibilities
- Analyze data and provide insights
- Recommend appropriate ML models
- Explain complex concepts in simple terms
- Suggest data collection improvements

# Communication Style
- Be precise and evidence-based
- Use examples to illustrate concepts
- Ask clarifying questions when assumptions might be wrong
- Acknowledge uncertainty when it exists

# Constraints
- Don't make recommendations without sufficient data
- Always consider ethical implications of data use
- Suggest validation approaches for any analysis
''';
```

### Common Pitfalls to Avoid

1. **Too Vague**: "Be helpful" → "Provide step-by-step solutions with explanations"
2. **Conflicting Instructions**: Don't give contradictory guidance
3. **Too Long**: Keep system prompts focused and concise
4. **Overly Restrictive**: Leave room for natural conversation flow

## Testing System Prompts

Validate your system prompts with different scenarios:

```dart
Future<void> testSystemPrompt(String systemPrompt, List<String> testQueries) async {
  final agent = Agent('openai:gpt-4o-mini', systemPrompt: systemPrompt);
  
  for (final query in testQueries) {
    print('Query: $query');
    final result = await agent.run(query);
    print('Response: ${result.output}');
    print('---');
  }
}

void main() async {
  final testQueries = [
    'Hello, who are you?',
    'Can you help me with a math problem?',
    'What should I do if I\'m confused?',
  ];
  
  await testSystemPrompt(
    'You are a patient math tutor who explains concepts step by step.',
    testQueries,
  );
}
```
