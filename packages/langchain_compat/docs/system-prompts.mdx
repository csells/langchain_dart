# System Prompts

Set default behavior and personality for your agents.

## Basic Usage

```dart
final agent = Agent(
  'openai',
  systemPrompt: 'You are a helpful math tutor. Show work step by step.',
);

final result = await agent.send('What is 15 * 23?');
// Response includes detailed steps
```

## Override with History

```dart
final agent = Agent(
  'openai',
  systemPrompt: 'You are a helpful assistant.', // Default
);

// Override with system message
final result = await agent.send(
  'Hello!',
  history: [
    ChatMessage.system('You are a pirate. Speak like one.'),
  ],
);
// Response: "Ahoy, matey!"
```

## System Prompt Templates

```dart
// Math tutor
const mathTutor = '''
You are an expert math tutor. When solving problems:
1. Show each step clearly
2. Explain the reasoning
3. Use simple language
4. Encourage students
''';

// Code reviewer
const codeReviewer = '''
You are a senior software engineer. When reviewing code:
1. Focus on correctness and readability
2. Suggest improvements with examples
3. Be constructive
4. Consider security
''';

// Dynamic prompts
String createExpert(String domain) => 
  'You are an expert in $domain. Provide clear, practical advice.';
```

## Precedence Rules

1. **Default**: Uses `systemPrompt` parameter
2. **Override**: System message in history overrides default
3. **Clean**: System messages excluded from response

```dart
final agent = Agent('openai', systemPrompt: 'Be concise.');

// Uses default
final r1 = await agent.send('Hello');

// Override with history
final r2 = await agent.send('Hello', history: [
  ChatMessage.system('Be verbose and detailed.'),
]);
```

## Provider Strengths

- **OpenAI**: Complex instructions, role-playing
- **Anthropic**: Detailed guidelines, consistency
- **Google**: Creative tasks, multimodal analysis
- **Mistral**: Multilingual, European context

## Best Practices

### Structure

```dart
const goodPrompt = '''
# Role
You are a [specific role].

# Key Tasks
- Task 1
- Task 2

# Style
[How to communicate]

# Constraints
[What not to do]
''';
```

### Tips

1. **Be specific** not vague
2. **Set boundaries** clearly
3. **Include examples** when helpful
4. **Keep it focused** and concise
